Blarggテストのソースコードから、以下の動作が期待されています：

1000サイクル以内 ： タイマー割り込みが発生してはいけない
1500サイクル以内 ： タイマー割り込みが発生すべき

set_test 4,"Timer doesn't work"
wreg TAC,$05
wreg TIMA,0
wreg IF,0
delay 500        ; 500サイクル待機
lda  IF
delay 500        ; さらに500サイクル待機
and  $04
jp   nz,test_failed  ; 1000サイクル後に割り込みがあってはダメ
delay 500        ; さらに500サイクル待機
lda  IF
and  $04
jp   z,test_failed   ; 1500サイクル後には割り込みがあるべき

なお、boot romをスキップして実行する仕様にしているため初期値に問題がある可能性がある。
boot ROMはオープンな実装のものを利用しているため、これを利用しても正しい初期値になる保証はない
また、external/Pyboy以下にはPyboyの実装が入っており、こちらはBlarggテストが通過することがわかっている。
